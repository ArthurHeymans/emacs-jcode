#+title: jcode

Emacs frontend for [[https://github.com/1jehuang/jcode][jcode]] using the Agent Client Protocol backend exposed by =jcode acp=.

This is currently an early direct-ACP implementation with a Pi-like two-buffer layout:

- read-only rendered chat buffer
- separate prompt composition buffer
- prompt history with =M-p= / =M-n=
- project file references: type =@= at a word boundary to choose a project file
- path completion via =TAB= for =./=, =../=, =~/=, and =/= paths
- cancel and disconnect commands
- attach/resume by session id, using ACP-compatible =cwd= and prompt content blocks
- completion picker and tabulated list for previous jcode sessions from =~/.jcode/sessions=
- =jcode-current= to reconnect to the latest session for the current project
- Pi-like header line with current session/status/model/project metadata
- tree-sitter markdown rendering for chat buffers via =md-ts-mode= when available, with hidden markup like Pi
- native jcode-inspired chat palette: off-white body text, amber headings, gray code blocks, blue links, and dim metadata
- collapsible tool blocks with short previews; press =TAB= or click the indicator to expand/collapse
- previous-session opening replays history by default so old chats are visible
- =session/update= rendering for assistant/user chunks and tool updates
- TRAMP-aware ACP process launching via =start-file-process=
- focused ERT coverage for ACP parameter shapes and rendering

* Requirements

- Emacs 29.1+
- =md-ts-mode= for tree-sitter markdown rendering.  If unavailable, chat buffers fall back to plain read-only rendering.
- =jcode= in =exec-path=, or set =jcode-program=

* Usage

Load the package, then run:

#+begin_src emacs-lisp
(require 'jcode)
#+end_src

Commands:

- =M-x jcode= creates or focuses a jcode session for the current project.
- =C-u M-x jcode= picks an existing session id and loads it.
- =M-x jcode-current= resumes the latest known session for the current project.
- =C-u M-x jcode-current= resumes the globally latest known session.
- =M-x jcode-connect=, also =M-x jcode-attach=, connects to a session through the native socket and requests takeover for live updates.
- =M-x jcode-reconnect= from a jcode chat/input reconnects the current session with native takeover, useful after another UI client owns the stream.
- =M-x jcode-list= opens a tabulated previous-session list.  Press =RET= to load with history replay or =r= to resume without replay.
- =M-x jcode-resume= picks a session id and resumes without full replay.
- =C-u M-x jcode-resume= loads with history replay.
- =M-x jcode-plan= opens the implementation plan.

Chat and input buffers show a compact header with session title/id, status, model, and project directory.  Keybindings remain available via mode maps, but are not shown in the header.

Input buffer keys:

- =C-c C-c= send prompt
- =C-c C-k= cancel active response
- =C-c C-d= disconnect Emacs client
- =M-p= / =M-n= prompt history
- =TAB= complete file references/paths
- =C-c C-s= steer current response while busy
- Type =@= at a word boundary to choose a project file and insert =@path/to/file=

Chat buffer keys:

- =C-c C-k= cancel active response
- =C-c C-d= disconnect Emacs client
- =TAB= expand/collapse a tool block at point
- =q= quit window

* Development

#+begin_src sh
nix develop
emacs -Q --batch -L . -L test -l test/jcode-test.el -f ert-run-tests-batch-and-exit
nix flake check
#+end_src

The flake exposes:

- =packages.default=: the Emacs package
- =packages.jcode=: a pinned build of upstream jcode from =github:1jehuang/jcode=
- =apps.jcode=: run the packaged jcode binary
